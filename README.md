# Farhad Blog

Учебный блог на Next.js. Главная страница показывает три статьи-заглушки и
демонстрирует разделение Server и Client Components.

## Рендеринг

Главная страница `app/page.tsx` является Server Component и использует ISR
(Incremental Static Regeneration): `export const revalidate = 30`.

Страница может быть заранее сгенерирована и закеширована. Через 30 секунд её
кеш считается устаревшим, а при следующем запросе Next.js создаёт свежую
версию в фоне. Для блога это хороший компромисс: контент быстро открывается,
но блок «Latest update» не остаётся старым надолго. Значение 30 секунд выбрано
для наглядной проверки механизма во время разработки; в настоящем блоге его
обычно увеличивают, если публикации меняются реже.

## Интерактивность

`src/components/LikeArticleButton/index.tsx` — Client Component. В нём есть
директива `"use client"`, поэтому компонент может использовать `useState` и
обновлять счётчик лайков в браузере. Кнопка подключена к каждой из трёх статей.

## Ошибка без `"use client"`

Для практики можно временно удалить первую строку `"use client"` из
`LikeArticleButton`. Next.js выдаст ошибку: React Hook `useState` нельзя
использовать в Server Component. Исправление — вернуть директиву первой
строкой файла. Она изолирует клиентский JavaScript только в кнопке, не делая
всю страницу клиентской.

## Запуск

```bash
npm run dev
```
